## 🔐 암호학 기본 개념

| 용어 | 설명 |
|------|------|
| **평문 (Plaintext)** | Clear Text (원본 데이터) |
| **암호문 (Ciphertext)** | 키로 암호화된 데이터 |
| **암호화 (Encryption)** | 평문 → 암호문 |
| **복호화 (Decryption)** | 암호문 → 평문 |
| **암호 해독 (Cryptanalysis)** | 암호화 알고리즘 분석/공격 |
| **키 (Key)** | 암호화/복호화에 사용되는 값 |
| **암호시스템 (Cryptosystem)** | 암호화 알고리즘 + 키의 조합 | 

### 양방향 암호 알고리즘

- **대칭키**: 암호화 키 == 복호화 키
- **공개키**: 암호화 키 ≠ 복호화 키

### 단방향 암호 알고리즘

**Hash** (키 없음, 복호화 불가능)

#### 1️⃣ 암호학적 해시함수
- MD5 (128bit)
- SHA-1 (160bit), SHA-224, SHA-256, SHA-384, SHA-512
- 목적: 해시함수의 충돌성 방지

#### 2️⃣ 비암호학적 해시함수
- CRC, Checksum, FCS

## 🔑 대칭키 암호 알고리즘

암호화 키 == 복호화 키 | 속도 빠름

| 분류 | 설명 | 예시 |
|------|------|------|
| **블록 암호** | 32bit, 64bit, 128bit 단위로 암호화 | DES, AES, IDEA (해외) / SEED, ARIA, LEA, HIGHT (국내) |
| **스트림 암호** | 1 bit, 1 byte씩 암호화 | RC4 |

---

## 🔓 공개키 (비대칭키) 암호 알고리즘

암호화 키 ≠ 복호화 키 | 속도 느림

| 분류 | 알고리즘 |
|------|---------|
| **소인수분해 기반** | RSA, RABIN |
| **이산대수 기반** | ECC, Elgamal, DSA, Diffie-Hellman (키교환프로토콜) |

## 🔄 하이브리드 암호 시스템

**대칭키 암호 + 공개키 암호의 조합**

- 대칭키의 빠른 속도 + 공개키의 보안성
- P/T → (대칭키 암호) → (공개키 암호) → C/T
---

## ✍️ 인증서와 전자서명

- **CA (공인인증기관)**: 공개키의 소유자를 증명하는 기관
- **공인인증서 / 공동인증서**: CA의 개인키로 서명한 전자문서
- **용도**: "이 공개키는 이 사람의 것이다"를 증명

## 🛡️ CIA 보안 3요소

| 요소 | 정의 | 위협 | 대응 |
|------|------|------|------|
| **Confidentiality<br>(기밀성)** | 허가된 사람만 정보 접근 | Sniffing, Eavesdropping (도청) | Encryption (암호화) |
| **Integrity<br>(무결성)** | 정보의 수정/삭제/변조 방지 | Malware, Modification | Hash Function (무결성 검증) |
| **Availability<br>(가용성)** | 필요할 때 시스템/정보 접근 | DoS/DDoS | Firewall, IDS/IPS |
